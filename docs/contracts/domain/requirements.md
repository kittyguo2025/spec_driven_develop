## 1. 项目概述

建设集团级**城建建筑档案管理系统**，私有云部署，支持多分公司数据隔离，管理建设工程与单体建筑档案的全生命周期，覆盖纸质数字化、BIM、CAD 竣工图及电子竣工包。

---

## 2. 已确认约束

| 项 | 值 |
|:---|:---|
| 部署 | 私有云（Docker / K8s） |
| 后端 | Java 17 + Spring Boot 3 |
| 前端 | Vue 3 + TypeScript |
| 外部集成 | 无 |
| 合规 | GB/T 50328-2019、DA/T 70-2018、DA/T 58-2014、DA/T 18-2022 |
| 销毁审批 | 分公司领导 + 分公司管理员（**无需集团管理员终审**） |

## 3. 用户角色与场景

### 3.1 集团管理员（group_admin）

- 管理全宗配置、分类号表版本、集团级制度参数
- 管理分公司（branch_org）信息
- 跨分公司统计报表（只读聚合）
- **不可**直接修改分公司已归档案卷内容

### 3.2 分公司管理员（branch_admin）

- 本分公司用户与角色分配
- 本分公司档案全流程：著录、上传、分类、借阅管理
- 发起销毁鉴定；与分公司领导共同确认销毁
- 本分公司统计与导出

### 3.3 普通业务人员（business_user）

- 本分公司建设项目/单体建筑档案著录
- 上传扫描件、CAD、BIM、竣工电子包
- 检索、浏览（权限范围内）
- 发起借阅申请
- **不可**审批、不可销毁、不可改 RBAC

### 3.4 公司领导（branch_leader）

- 本分公司借阅、密级变更、开放鉴定、销毁审批
- 本分公司高层统计看板（只读）
- **不可**跨分公司；**不可**直接修改著录项

## 4. 功能需求

| ID | 需求 | 优先级 |
|:---|:-----|:------:|
| FR-01.1 | 支持集团—分公司两级组织 | P0 |

| ID | 需求 | 优先级 |

| FR-02.1 | 管理建设项目基本信息（对齐 GB/T 50328） | P0 |

## 5. 非功能需求

### NFR-01 性能

| ID | 需求 | 指标 |
|:---|:-----|:-----|
| NFR-01.1 | 列表检索响应 | P95 < 2s（10 万级条目） |
| NFR-01.2 | 单文件上传 | 支持 ≥ 2GB（BIM/CAD） |
| NFR-01.3 | 并发用户 | ≥ 200（私有云可水平扩展） |

### NFR-02 安全

| ID | 需求 |
|:---|:-----|
| NFR-02.1 | 传输 HTTPS；存储加密可选（私有云 KMS） |
| NFR-02.2 | 密码策略、会话超时、登录失败锁定 |
| NFR-02.3 | RBAC + 分公司隔离；越权访问拒绝并审计 |
| NFR-02.4 | 上传病毒扫描 |

### NFR-03 可用性与部署

| ID | 需求 |
|:---|:-----|
| NFR-03.1 | 私有云 Docker Compose 开发；K8s 生产 |
| NFR-03.2 | 配置经环境变量注入，无硬编码密钥 |
| NFR-03.3 | 目标可用性 99.5%（私有云 SLA 内） |

### NFR-04 合规

| ID | 需求 |
|:---|:-----|
| NFR-04.1 | 著录、术语、检测符合第四节所列国标 |
| NFR-04.2 | 已归档数据禁止物理 DELETE |
| NFR-04.3 | AI 开发遵循 `ai-agent-boundaries.md` |

### NFR-05 可维护性

| ID | 需求 |
|:---|:-----|
| NFR-05.1 | 契约驱动：OpenAPI + JSON Schema 为 API/数据真相源 |
| NFR-05.2 | domain-core 零 Spring 依赖，可独立单元测试 |
| NFR-05.3 | 契约校验 CI 门禁 |

## 6. 不在范围内（Out of Scope）

- 外部 GIS、政务门户、OA、SSO 对接
- 外部 BIM 协同平台实时同步
- 移动端 App（首期仅 Web）
- 纸质档案实体库房 RFID/温湿度 IoT

---

## 7. 验收标准摘要

1. 电子竣工包按 GB/T 50328 目录导入 E2E 成功
2. DA/T 70 四性检测失败文件无法归档
3. 跨分公司越权访问全部拒绝
4. 销毁流程经分公司领导+管理员双审，无集团终审环节
5. 审计日志 INSERT-only 验证通过
