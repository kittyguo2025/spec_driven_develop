# 领域术语表（Glossary）

> **标准依据**: DA/T 58-2014《电子档案管理基本术语》为主；GB/T 50328-2019、DA/T 18-2022 补充  
> **用途**: UI 文案、API 字段描述、代码注释、数据库注释的唯一术语来源  
> **规则**: 禁止自造词替代本表术语（AI Hard Deny HD-12 关联）

---

## 1. 档案一般术语

| 术语 | 英文标识 | 定义 | 标准来源 |
|:-----|:---------|:-----|:---------|
| 档案 | archive | 国家机构、社会组织以及个人从事政治、军事、经济、科学、技术、文化、宗教等活动直接形成的对国家和社会具有保存价值的各种文字、图表、声像等不同形式的历史记录 | DA/T 58 |
| 电子档案 | electronic archive | 通过计算机等电子设备形成、办理、传输和存储的数字格式的各种信息记录 | DA/T 58 |
| 全宗 | fonds | 一个立档单位在其活动过程中形成的各种形式档案有机整体，是档案库分类和管理的基本单位 | DA/T 58 |
| 案卷 | volume | 由互有联系的若干文件组合而成的档案保管单位 | DA/T 58 |
| 文件 | document / item | 归档保存的单份或组合记录；本系统著录层称「件」 | DA/T 58 |
| 件 | archive_item | 案卷内的单份归档文件，最小著录单元之一 | DA/T 18 |
| 目录 | catalogue | 按照一定规则组织条目并指示档案出处与存址的检索工具 | DA/T 58 |
| 元数据 | metadata | 描述档案内容、结构、背景及其管理过程的数据 | DA/T 58 |
| 著录 | description | 对档案内容和结构信息进行的描述过程 | DA/T 18 |
| 档号 | archival code | 以字符形式赋予档案的固定唯一标识，反映分类与排列顺序 | DA/T 18 |
| 保管期限 | retention period | 档案在保存期满后仍需继续保存或可以销毁的时间规定 | GB/T 50328 |
| 密级 | security classification | 档案保密等级 | DA/T 58 |
| 开放 | access_open | 档案依法向社会或特定范围提供利用 | DA/T 58 |
| 鉴定 | appraisal | 对档案价值、保管期限、开放范围或销毁进行的审查判断 | DA/T 58 |
| 销毁 | destruction | 经鉴定无保存价值且审批通过后消除档案载体的行为 | DA/T 58 |
| 借阅 | borrow | 档案利用者按规定程序临时借出档案的行为 | 业务扩展 |
| 利用 | access | 对档案内容的查考、摘录、复制等活动 | DA/T 58 |

## 2. 电子档案术语

| 术语 | 英文标识 | 定义 | 标准来源 |
|:-----|:---------|:-----|:---------|
| 真实性 | authenticity | 电子档案内容与形成过程可信，来源可确认 | DA/T 70 |
| 完整性 | integrity | 电子档案内容、结构、背景信息齐全，未被非法改动 | DA/T 70 |
| 可用性 | usability | 电子档案可被正常读取、检索与利用 | DA/T 70 |
| 安全性 | security | 电子档案免受未授权访问、篡改或破坏 | DA/T 70 |
| 四性检测 | four-properties check | 对电子档案真实性、完整性、可用性、安全性的检测 | DA/T 70 |
| 电子文件 | electronic document | 尚未归档或正在办理中的数字格式文件 | DA/T 58 |
| 格式信息 | format information | 描述电子文件编码、版本、结构的数据 | DA/T 58 |
| 捕获 | capture | 将电子文件及其元数据纳入档案系统的过程 | DA/T 58 |
| 封装包 | encapsulation package | 将电子文件、元数据、校验信息打包的结构化单元 | GB/T 50328 |
| 数字摘要 | digest / hash | 用于验证文件完整性的哈希值（本系统采用 SHA-256） | DA/T 70 |
| 长期保存格式 | long-term preservation format | 适合长期存阅的文件格式（如 PDF/A、OFD） | DA/T 47 参考 |

## 3. 建设工程归档术语（GB/T 50328）

| 术语 | 英文标识 | 定义 | 标准来源 |
|:-----|:---------|:-----|:---------|
| 建设工程 | construction project | 土木工程、建筑工程、线路管道、设备安装及装修工程等新、改、扩建活动 | GB/T 50328 |
| 工程文件 | project document | 工程建设过程中形成并应归档的文件 | GB/T 50328 |
| 竣工图 | as-built drawing | 工程竣工后反映最终施工结果的图样 | GB/T 50328 |
| 电子竣工包 | electronic completion package | 按 GB/T 50328 目录结构组织的竣工电子文件集合 | 本系统 |
| 单体建筑 | building unit | 建设项目下可独立标识的单栋建筑物 | 本系统 |
| 立卷 | volume filing | 将文件按规则组合成案卷的过程 | GB/T 50328 |
| 归档 | filing / archive filing | 文件经整理后移交档案部门保存的过程 | GB/T 50328 |

## 4. 数字资产扩展术语（BIM/CAD）

| 术语 | 英文标识 | 定义 | 说明 |
|:-----|:---------|:-----|:-----|
| 数字资产 | digital_asset | 系统管理的二进制文件实体（扫描件、CAD、BIM 等） | 本系统契约 |
| BIM 模型 | bim_model | 建筑信息模型文件（如 IFC、RVT） | digital_asset 子类型 |

| 预览格式 | preview_format | 便于在线浏览的派生格式 | 可选生成 |

## 5. 组织与权限术语

| 术语 | 英文标识 | 定义 | 说明 |
|:-----|:---------|:-----|:-----|
| 集团 | group | 顶层组织，管理多个分公司 | 本系统 |
| 分公司 | branch_org | 数据隔离与权限边界单元 | 对应 `branch_org_id` |
| 集团管理员 | group_admin | 全集团配置与跨分公司统计权限 | 四类角色之一 |
| 分公司管理员 | branch_admin | 本分公司用户与档案全流程管理 | 四类角色之一 |
| 业务人员 | business_user | 本分公司著录、上传、检索、发起申请 | 四类角色之一 |
| 公司领导 | branch_leader | 本分公司审批与高层只读统计 | 四类角色之一 |

## 6. 状态术语（与 state-machines.yaml 对齐）

| 术语 | 英文标识 | 含义 |
|:-----|:---------|:-----|
| 草稿 | draft | 著录中，未提交 |
| 待检测 | pending_check | 已上传，等待四性检测 |
| 检测失败 | check_failed | 四性检测未通过 |
| 已归档 | archived | 检测通过并正式入库 |
| 已锁定 | frozen | 不可修改著录项 |
| 借出 | borrowed | 借阅中 |
| 鉴定中 | appraising | 开放/销毁/密级鉴定流程中 |
| 已销毁 | destroyed | 逻辑销毁（保留审计与销毁记录） |
| 已删除 | deleted | 逻辑删除（仅草稿或未归档） |

## 7. 禁止使用的非标准称谓

| 禁止使用 | 应使用 |
|:---------|:-------|
| 文件夹 | 案卷 |
| 文档库 | 档案 |
| 删除档案 | 销毁（已归档）/ 逻辑删除（未归档） |
| 永久删除 | 销毁（须审批） |
| 公司 | 分公司（指 branch_org 时） |
| 用户组 | 角色（RBAC） |
