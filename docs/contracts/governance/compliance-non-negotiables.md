# Compliance Non-Negotiables — 不可妥协合规红线

> 以下条款在任何实现、AI 生成代码、或紧急修复中**均不可降级或绕过**。

## 1. 适用标准

| 标准 | 适用范围 |
|:-----|:---------|
| GB/T 50328-2019 | 建设工程文件归档范围、目录、保管期限 |
| DA/T 18-2022 | 档案著录规则与元数据 |
| DA/T 58-2014 | 电子档案管理术语（UI/文档/代码命名） |
| DA/T 70-2018 | 文书类电子档案四性检测 |

## 2. 数据不可破坏

- 已归档实体：**禁止物理 DELETE**
- 审计日志：**禁止 UPDATE / DELETE**（仅 INSERT）
- 销毁：仅通过审批工作流，记录销毁凭证与审批链

## 3. 入库门禁

- 所有电子文件（扫描件、CAD、BIM、竣工包内文件）入库前**必须通过** DA/T 70 检测流水线
- 检测失败：**不得**进入 `archived` 状态
- 检测项清单以 `da-t70-checklist.yaml` 为准，不可运行时删减

## 4. 多租户隔离

- 除集团管理员外，所有读写操作必须限定 `branch_org_id`
- 公司领导审批范围：本分公司待办 only

## 5. 术语一致

- 用户可见文案、数据库注释、API 字段描述必须使用 `glossary.md` 定义术语
- 禁止自造词替代国标术语（如用「文件夹」替代「案卷」）

## 6. 审批不可跳过

- 借阅、销毁、密级变更、开放鉴定：必须完整走 `state-machines.yaml` 定义路径
- 不允许「一键通过」API 或管理后门

## 7. 长期保存

- 电子文件保留原始格式 + SHA-256 哈希
- 格式白名单见 `digital-asset.schema.json`（Phase 2 产出）

## 8. 冲突解决

国标 > 集团制度 > 本系统契约 > 实现代码

若实现与契约冲突，**以契约为准并修复实现**。
